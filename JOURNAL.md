# Журнал работ

## 2026-06-09 | Этап 1. Каркас
- **Результат:** Собран каркас проекта. Реализован CLI-интерфейс для приема пути к папке через `argparse` с валидацией. Создан модуль `db.py` для инициализации SQLite. При запуске автоматически создается папка `data/` и база данных со схемой таблицы `files`.
- **Проблемы:** Нет.
- **Следующий шаг:** Этап 2. Рекурсивное сканирование папки и сохранение метаданных файлов в БД.

## 2026-06-10 | Этап 2. Сканирование
- **Результат:** Реализован рекурсивный обход папки через `os.walk`. Создан модуль `scanner.py` со сбором метаданных. Добавлен фильтр по расширениям (`--ext`) и автоматическое игнорирование скрытых системных файлов/папок. Настроена умная синхронизация с БД: новые файлы добавляются, измененные обновляются, удаленные из папки — стираются из БД.
- **Проблемы:** Нет.
- **Следующий шаг:** Этап 3 (Отчеты, статистика и поиск дубликатов по хэшу).

## 2026-06-11 | Этап 3. Дубликаты
- **Результат:** Реализовано вычисление хэша содержимого файлов (MD5) через `hashlib` с поблочным чтением. Создан модуль `hasher.py` с оптимизацией: хэш пересчитывается только если файл новый или изменился (сравнение size/mtime). Хэши сохраняются в SQLite. Добавлен CLI-флаг `--find-dupes` для поиска и вывода групп файлов с одинаковым хэшем.
- **Проблемы:** Нет.
- **Следующий шаг:** Этап 4. Резервная копия

## 2026-06-12 | Этап 4. Резервная копия
- **Результат:** Реализован модуль `backup.py` для сравнения проиндексированного источника с папкой бэкапа. Сравнение идет по относительным путям, размеру и хэшу содержимого. Добавлены таблицы `backup_checks` и `backup_diffs` для сохранения истории всех проверок в SQLite. В CLI добавлен флаг `--backup <путь>`, который формирует и выводит подробный итоговый отчет (отсутствующие, измененные, лишние файлы).
- **Проблемы:** Нет.
- **Следующий шаг:** Защита
